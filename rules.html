🏭 交貨單稽核系統 - 規則說明

本工具主要用於自動檢查中機輥輪再生交貨單中的 KEY 錯字、算錯數、漏填資料，確保資料準確度並提升覆核效率。

📂 資料提供方式

方式說明
📸 拍照上傳拍攝紙本單據或掃描 PDF，AI 自動讀取文字（約 30-60 秒，超量產生些微 API 費用）。
📊 Excel 上傳直接丟入電子檔，速度最快（1 秒且免費）、準確度最高（表格直讀）。
📂 JSON 上傳用於讀取舊有的系統紀錄檔案。
☁️ 巨集一鍵稽核系統已部署至 GCP Cloud Run，可直接在 Excel 檔案內利用巨集一鍵執行稽核，完全不用離開 Excel 頁面。

1. 📋 表頭資訊稽核 (Header Audit)

檢查項目邏輯與條件
🚨 嚴重混單同一份文件中偵測到多種不同的工令單號或工作申請單編號出現在同一份文件中。
⚠️ 存放位置錯誤依據工令前綴自動判定廠區。例如：W3 開頭必須儲存於 W3 SHOP、W2 開頭為 W2 SHOP、Y461 開頭為 Y461 SHOP、OW 則須為 中龍 SHOP。
⚠️ 工令格式錯誤單號不符合預設規則（須為 W, R, O, Y 開頭且長度為 10 碼）。內部新增防雜訊機制：O 開頭須含至少 2 個數字，其餘須含至少 6 個數字，且嚴格排除包含 KEY/WAY 等雜訊字眼。
⚠️ 工作申請單編號異常除了檢查混單外，格式必須為 2 開頭，且長度精準限制為 10 碼或 12 碼。
📅 日期不一致與缺失整份文件出現兩個以上的預定交貨日或實際交貨日；或是全卷皆未填寫或無法辨識日期。
⏰ 逾期交貨任一項目的「實際交貨日」晚於「預定交貨日」，系統將自動計算並回報確切延遲天數。
📑 頁碼重複偵測到相同的頁碼數字於全卷中出現兩次以上（防止重複影印或掃描）。
📑 頁碼缺漏/未從頭開始嚴格檢查系統擷取到的實體頁碼是否從第 1 頁開始，且必須連續無中斷。

2. 📏 規格與長約稽核 (Spec & Contract Audit)

檢查項目邏輯與條件
🛑 長約項次比對系統會自動讀取 contract_items.xlsx 執行 100% 嚴格字串比對（經過去除空格、特殊符號後的英數中完美配對）。若完全無法在長約清單中找到吻合項目，即判定為非長約項目並報錯（標題含「報廢」之項目自動豁免此項核對）。
⚠️ 規格取代落差當系統透過 rules.xlsx 強制更換規格後，若「原始辨識內容」與「強制更換規格」相似度低於 100%（系統比對時會自動保留 "MM" 等關鍵單位以確保精準），會觸發異常提醒人工覆核。

3. 🔢 工程與數值稽核 (Numerical Audit)

檢查項目邏輯與條件
⚠️ 規格缺漏/無效偵測到項目名稱，但規格欄位為空；或該項目屬於必須嚴格檢查數值之分類，但規格內容解析不出任何有效數字基準。
🛑 單位異常 (標題括號)標題後方括號內除數字外，單位僅允許填寫「PC」或「SET」，若包含其他文字雜訊將觸發報錯。
🛑 數據格式錯誤/缺漏數值欄位填寫了 N/A、OK、- 或文字雜訊。若明細完全未填寫任何數據，將觸發「數據缺漏」攔截。
🛑 數值異常 (不可為零)全域防呆限制，實測數據絕對不可填寫 0 或 0.0，若無實際數值應填寫 N/A 或是保持空白。
🛑 單位異常 (整批製程)熱處理：數量後方必須標示單位「KG」。
研磨/動平衡：數量後方必須標示單位「IN2」或「IN²」。
*(若標題含校正、串列式、W3X、內孔、小鋼胚等白名單關鍵字，則豁免單位檢查)*
🛑 明細異常 (反向檢查)熱處理：僅需填寫總重量，底下的「編號與尺寸」明細欄位必須保持空白。
研磨/動平衡：除了總量與單位，底下的明細必須逐筆填寫。
🛑 報廢異常若標題包含「報廢」，其尺寸明細必須填寫為 N/A 或保留空白，嚴禁填寫實際數值。
🛑 豁免異常 (空值與純數字)系統強制規定被判定為「豁免」或「SKIP」之項目：
1. 不可為空：對應欄位必須有文字註記（如 N/A、OK）。
2. 不可純數字：避免現場誤填為實際尺寸。
🛑 尺寸異常分類檢驗依不同分類執行防呆演算法：
• 銲補 (Min Limit)：須為純整數，低於下限或超過上限 20 以上即報錯。
• 精加工/研磨 (Range)：必須填寫兩位小數，且精準落在公差區間內。
• 幾何/形位公差：不可為負數，須填兩位小數且不超過上限。
• 1機310軸頸特規：同編號兩端尺寸差值必須大於等於 8mm。
• 驅動輥內孔階梯特規：自動檢查數值集中度（差值 < 1mm），限制密集上限。
🛑 特規異常 (本體車修)針對上限小於等於 150mm 的本體車修項目，若實測值小於下限，頁面備註必須精確包含「依現場需求調整車修尺寸下限」，否則報錯。
🛑 數據損壞與單格豁免偵測到 [!] 壞軌標記將直接報錯。針對整批項目，若單一數值欄位填寫了數量單位（如 IN, SET），該格自動豁免嚴格數值檢驗。

4. 🧮 會計與數量匯總稽核 (Accounting Audit)

檢查項目邏輯與條件
⚠️ 申請與實交差異總表內「申請數量」不等於「實交數量」，或兩者皆為 0 / 空白。
🛑 單位異常(總表)總表單位的嚴格核對：熱處理必須為 KG；研磨/動平衡必須為 IN2；其餘常規項目為 PC 或 SET，絕對不可留空。
🛑 統計不符(單項)明細標題要求的目標數與實際抄錄有數據之筆數不符（自動結合計價倍率轉換）。
⚠️ 編號重複同一個工件 ID 在本體清單重複出現 2 次以上，或在軸頸/鍵槽清單重複出現 3 次以上。
🛑 明細匯總不符總表實交數不等於明細項目經過「運費比重」、「聚合規則 (u_agg)」加總後的總和。
👻 幽靈明細明細項目名稱無法匹配至總表中的任何一項。
🛡️ 語意防撞過濾系統配對時嚴格防撞，自動隔離「新品」與「舊品」之項目，避免會計項目錯配。
🌊 運費階層與流道獨立流道：「新品組裝」直接計入運費。
瀑布篩選：舊品拆裝 > 本體未再生 > 本體銲補 > 一般銲補 > 面研磨 > 動平衡。

5. 🔄 流程與溯源稽核 (Process Audit)

檢查項目邏輯與條件
🛑 工序尺寸合理性透過履歷交叉比對：
1. 尺寸倒置：例如粗車尺寸小於精車尺寸。
2. 差值檢驗：本體銲補與再生差值須介於 4~10mm(BCC 350 PPINCH ROLL**5mm ~ 10mm** 之間)；軸頸銲補與再生差值須介於 3~7mm。
🛑 身分衝突同一個零件編號「嚴禁」同時出現在「新品組裝」與「舊品拆裝」明細中。
申報為「新品」者，嚴禁具備「粗車」紀錄，但在履歷溯源時會自動豁免前置工序斷層。
🛑 流道異常(防撞)終極防撞演算法：同一工序階段中，不同項目嚴禁認領相同的編號（白名單配對除外），防止複製貼上錯誤。若出現部分交集錯位，主動抓出跨流道誤填。
🛑 拆裝池分流嚴格區分「一般成人池」與「專屬兒童池（如 W3#1~3機）」，兩者禁止跨池對應。
🛑 工序與相依有軸頸加工卻無本體紀錄；有銲補卻缺失再生紀錄；軸頸項目強制必須包含「未再生、銲補、再生」完整履歷。
🛑 CPLG/DRIVEN 異常頭尾屬性不一致，或名稱標註 CPLG 但缺乏 VIP 關鍵字。CPLG 軸頸規格內禁止出現「非」字。

6. ☁️ 雲端紀錄與例外處理

檢查項目邏輯與條件
✅ 雲端同步交貨單檢查合格紀錄會自動上傳至 Firebase 結構化資料庫，方便後續快速查詢。
🛑 開關控制系統支援偵測 STOP_LOGGING 開關（優先讀取 Cloud Run 環境變數），開啟後將直接阻擋寫入，方便除錯。
